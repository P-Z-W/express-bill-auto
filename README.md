# 毅播云仓内部管理平台 V4.0

基于 Flask 的云仓内部后台系统，当前已完成快递对账模块，支持局域网 Web 操作。

---

## 快速启动

双击 `start.bat` 启动服务，浏览器访问：

```
本机：http://127.0.0.1:5000
局域网：http://[本机IP]:5000
```

---

## 功能模块

### 快递对账（当前版本）

每月自动完成四步流程：

1. **订单下载** — 连接阿里云 MySQL，按日期范围导出订单数据
2. **账单合并** — 扫描 `data/YYYY-MM/` 自动识别申通/中通账单，清洗合并
3. **运单对账** — 运单号匹配回填团队，按省份+重量判断计费方式，读取报价计算金额
4. **账单拆分** — 按团队生成标准化 Excel，含汇总统计表

**Web 页面：**

| 路径 | 说明 |
|---|---|
| `/` | 首页看板：费用趋势、团队 TOP5、快递占比、环比涨跌 |
| `/run` | 运行页：上传账单、触发运行、实时日志进度 |
| `/history` | 历史记录：每月运行次数、结果、耗时、下载 |
| `/stats` | 统计报表：团队费用汇总、异常运单分析 |
| `/config` | 系统配置：报价表管理、运行参数调整 |

## 目录结构

```
yibo-backoffice/
├── app.py                  # 入口：注册 Blueprint，启动服务
├── start.bat               # 双击启动
├── main.py                 # CLI 模式入口（可独立运行）
│
├── core/                   # 基础设施
│   ├── cache.py            # Excel 缓存、接口结果缓存
│   ├── task_runner.py      # 任务调度、日志捕获、SSE 推送
│   └── utils.py            # 公共工具函数、openpyxl 样式常量
│
├── modules/
│   ├── express/            # 快递对账模块
│   │   ├── routes.py       # Blueprint 路由（全部接口）
│   │   ├── order_db.py     # 数据库订单下载
│   │   ├── merge_express.py    # 账单合并
│   │   ├── order_matching.py   # 运单对账匹配
│   │   └── split_bill_by_team.py  # 按团队拆分账单
│   ├── storage/            # 仓储费（占位）
│   └── finance/            # 财务汇总（占位）
│
├── config/
│   ├── settings.py         # 全局配置（月份、路径、SQL日期）
│   ├── price_config.xlsx   # 申通/中通报价 + 客户加收单价
│   ├── express_config.json # 快递公司配置
│   └── SQL-config.txt      # SQL 模板（不入版本库）
│
├── templates/              # 页面模板
├── data/YYYY-MM/           # 原始快递账单（不入版本库）
└── output/YYYY-MM/         # 输出结果（不入版本库）
```

## 每月操作流程

1. 双击 `start.bat` 启动服务
2. 浏览器打开运行页 `/run`
3. 上传申通/中通原始账单 Excel
4. 点击「开始运行」，实时查看日志
5. 完成后在历史记录页下载 zip

## 计费规则说明

| 计费方式 | 触发条件 |
|---|---|
| 全国均重 | 结算重量 ≤ 1kg，或 1–3kg 非新疆/西藏 |
| 单票 | 结算重量 ≥ 3kg |
| 新西1-3公斤 | 新疆/西藏 且 1kg < 重量 < 3kg |

特殊规则：新西 1kg 内订单数超全部订单 1% 才收费（`r11_value = 超出单数 × 10`）
